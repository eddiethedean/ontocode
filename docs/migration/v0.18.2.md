# Migration: v0.18.1 → v0.18.2

v0.18.2 is a **patch** on the Protégé Desktop parity gate: Windows path handling, reasoner cancel/`ops_lock`, Manchester cardinality, Find Usages / prefixes, Turtle literals, and extension panel restore. See [CHANGELOG](https://github.com/eddiethedean/ontocode/blob/main/CHANGELOG.md) and the v0.18 overview in [migration/v0.18.md](v0.18.md).

## What changed

### Windows / VS Code extension

- Normalize Rust `canonicalize()` `\\?\` paths before VS Code URI and path equality checks.
- Prefix Manager and inspector axiom patches write through normalized workspace document URIs.
- Remove duplicate `ontocode.showImportsPanel` registration so activation succeeds on all platforms.

### Reasoner

- Release `ops_lock` during classify so index/patch ops are not blocked; honor `$/cancelRequest` and skip snapshot updates on cancel ([#211](https://github.com/eddiethedean/ontocode/issues/211)).
- Panel running state, `lastRunAt` / `dirty`, focus sync, and explanation catalog fingerprint races ([#212](https://github.com/eddiethedean/ontocode/issues/212), [#219](https://github.com/eddiethedean/ontocode/issues/219), [#220](https://github.com/eddiethedean/ontocode/issues/220), [#221](https://github.com/eddiethedean/ontocode/issues/221)).

### Manchester / Turtle / Find Usages

- Property-first qualified cardinality parse + serialize; typed `xsd:nonNegativeInteger` Turtle emit ([#226](https://github.com/eddiethedean/ontocode/issues/226), [#231](https://github.com/eddiethedean/ontocode/issues/231)).
- Find Usages line guards, default-prefix CURIEs, annotation subject hits, and broader `prefixes_from_turtle` ([#225](https://github.com/eddiethedean/ontocode/issues/225), [#228](https://github.com/eddiethedean/ontocode/issues/228), [#232](https://github.com/eddiethedean/ontocode/issues/232), [#237](https://github.com/eddiethedean/ontocode/issues/237)).
- Turtle literal / undefined-prefix scanners for long strings and all quote forms ([#210](https://github.com/eddiethedean/ontocode/issues/210), [#214](https://github.com/eddiethedean/ontocode/issues/214), [#224](https://github.com/eddiethedean/ontocode/issues/224), [#233](https://github.com/eddiethedean/ontocode/issues/233)).

### Authoring safety / panels

- Workspace path guards and honest toasts for create/Manchester/inspector/imports ([#213](https://github.com/eddiethedean/ontocode/issues/213), [#230](https://github.com/eddiethedean/ontocode/issues/230), [#239](https://github.com/eddiethedean/ontocode/issues/239)).
- Imports / graph / Manchester session restore and explorer refresh after cancelled editor sync ([#227](https://github.com/eddiethedean/ontocode/issues/227), [#229](https://github.com/eddiethedean/ontocode/issues/229), [#235](https://github.com/eddiethedean/ontocode/issues/235), [#236](https://github.com/eddiethedean/ontocode/issues/236)).

## Breaking changes

None.

## Upgrade

1. Bump to OntoCore / OntoCode **0.18.2** (`cargo install ontocore-cli --locked --version 0.18.2`, or Marketplace / VSIX).
2. No CI assertion changes required beyond taking the new pin.
