# Migration: v0.19.0 → v0.20.0 (draft)

> **Status:** Draft for the in-progress **0.20.0** line. **0.20.0 is not released yet.** Latest tagged release remains **v0.19.0**.

## Summary

v0.20.0 is planned to harden Turtle patch apply/remove matching for common Protégé/ROBOT Turtle styles, and to deliver workspace runtime (EPIC-003). Patch operation names and wire formats are unchanged.

## What changed so far (on branch, unreleased)

### Turtle patch matching

- Language-tagged and typed literals match for `RemoveLabel` / `RemoveComment` / data-property removes
- Primary-type and characteristic checks ignore `a` / type substrings inside comments and strings
- IRI-object removes match both CURIE and `<IRI>` surface forms
- `SetOntologyIri` rewrites `rdf:type owl:Ontology` declarations in place
- Property-chain validation no longer invents colliding short-name CURIEs

### Packaging

- Workspace crates, CLI, LSP, and VS Code extension version set to **0.20.0** (development; not tagged)

## Still planned for v0.20

- Workspace runtime (EPIC-003 / BLOCKER_03): multi-ontology registry, dirty/save, session persistence, Save/Save All

## What did not change

- Semantic transaction apply path introduced in v0.19 (`ontocore-edit`)
- RDF/XML and OWL/XML write-back (still read-only; planned v0.21)

## Upgrade steps (when 0.20.0 is tagged)

1. Bump OntoCode extension and `ontocore` CLI to **0.20.0**
2. No migration of saved patch payloads required
