# Migration: v0.23.0 → v0.24.0

> **Status:** Applies when upgrading from **v0.23.0** to tagged **v0.24.0**.

## Summary

v0.24.0 ships **semantic services** (EPIC-006 / EPIC-007; `PAR-REF-001`, `PAR-QRY-002`): Protégé-style DL Query, richer refactoring (merge/replace, move axioms, ontology merge, flatten/cleanup imports, locality module extract, SWRL-aware IRI rewrite), workspace search LSP, and Query Workbench DL mode.

## What changed for editors

- Query Workbench gains **DL** mode: Manchester class expressions with Instances / Subclasses / Superclasses / Equivalents tabs (asserted or inferred)
- Refactor preview shows affected entity/axiom counts; SWRL rule JSON literals rewrite on rename/merge/replace
- New refactor commands: merge entities, replace entity, move axioms, merge ontologies, flatten imports, cleanup imports, locality extract
- Search Entities uses live `ontocore/search`

## New CLI / LSP

| Surface | Detail |
|---------|--------|
| CLI | `ontocore dl-query`, `ontocore refactor merge`, `refactor replace`, `refactor merge-ontologies`, `flatten-imports`, `cleanup-imports`, extract `--locality` |
| LSP | `ontocore/dlQuery`, `ontocore/search`; expanded `previewRefactor` request kinds |

## What did not change

- JSON-LD / TriG / N-Triples remain read-only
- Refactor transforms remain **Turtle-first**; RDF/XML, OWL/XML, and OBO are skipped with warnings (XML write-back still via patches)
- Full multi-format serializer-independent refactor remains a follow-on; see [Known limitations](../known-limitations.md)
- Undo/redo batch UX stays **v1.0**; full parity CI dashboard stays **v0.25**

## Upgrade steps

1. Bump OntoCode extension and `ontocore` CLI to **0.24.0**
2. Re-index open workspaces after upgrade
3. Prefer DL Query Workbench for Manchester class-expression queries (SQL/SPARQL modes unchanged)
