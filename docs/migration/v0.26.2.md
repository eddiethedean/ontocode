# Migration: v0.26.1 → v0.26.2

v0.26.2 is a **patch** on the Protégé Desktop behavioral test-port line: host focus/transaction integrity, semantic-diff/edit/query correctness, exhaustive XML/OWL IRI remap, aligned merge semantics across formats, and residual invert / SPARQL / orphan-diagnostic false positives. See [CHANGELOG](https://github.com/eddiethedean/ontocode/blob/main/CHANGELOG.md) and the v0.26 overview in [migration/v0.26.md](v0.26.md).

## What changed

### Extension host state ([#417](https://github.com/eddiethedean/ontocode/pull/417))

- `focusRelay.setFocus` rejects older timestamps so late panel posts cannot overwrite newer focus ([#402](https://github.com/eddiethedean/ontocode/issues/402))
- WorkspaceStore `navigationBack` / `navigationForward` sync inspector / graph / explorer entity slices ([#352](https://github.com/eddiethedean/ontocode/issues/352))
- `applyRefactor` registers self-writes (and marks reasoning dirty) like axiom patches ([#396](https://github.com/eddiethedean/ontocode/issues/396))
- `WorkspaceTransactionManager.begin` reserves `pending` before `await` to close the concurrent-begin race ([#386](https://github.com/eddiethedean/ontocode/issues/386))

### Engine residual correctness ([#417](https://github.com/eddiethedean/ontocode/pull/417))

- `RemoveSynonym` with a recorded scope inverts to `AddSynonym` ([#400](https://github.com/eddiethedean/ontocode/issues/400))
- SPARQL update detector skips `USING` / `USING NAMED` prologue like `WITH` ([#399](https://github.com/eddiethedean/ontocode/issues/399))
- `orphan_classes` treats Manchester / restriction parents as non-orphans ([#401](https://github.com/eddiethedean/ontocode/issues/401))

### Semantic diff, edit, and query integrity ([#419](https://github.com/eddiethedean/ontocode/pull/419))

- Axiom/annotation diff keys include `ontology_id` so multi-document removals are visible ([#395](https://github.com/eddiethedean/ontocode/issues/395))
- Rename detection requires **added** exact `owl:sameAs` / `skos:exactMatch` (not removals or substring matches) ([#389](https://github.com/eddiethedean/ontocode/issues/389))
- AllDifferent rewrite fails closed when a list member CURIE cannot expand ([#350](https://github.com/eddiethedean/ontocode/issues/350))
- `atomic_write` best-effort removes `.tmp` on mid-write I/O failure (OWL + OBO) ([#343](https://github.com/eddiethedean/ontocode/issues/343))
- `createOntology` rejects format/extension mismatches (e.g. `ttl` under `.owl`) ([#342](https://github.com/eddiethedean/ontocode/issues/342))
- `SELECT *` with zero rows still returns table schema columns ([#385](https://github.com/eddiethedean/ontocode/issues/385))
- SWRL rule list dedupes on document + canonical rule JSON ([#362](https://github.com/eddiethedean/ontocode/issues/362))

### Multi-format refactor IRI remap and merge ([#420](https://github.com/eddiethedean/ontocode/pull/420), [#421](https://github.com/eddiethedean/ontocode/pull/421))

- XML/OWL IRI remap covers all Horned `Component` kinds and recursive `DataRange` ([#368](https://github.com/eddiethedean/ontocode/issues/368), [#376](https://github.com/eddiethedean/ontocode/issues/376))
- Entity merge uses **delete-then-remap** across Turtle, OBO, and RDF/XML/OWL/XML — merge-owned labels/parents are dropped instead of absorbed onto the survivor ([#369](https://github.com/eddiethedean/ontocode/issues/369))

## Breaking changes

None.
