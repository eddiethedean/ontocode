# Migration: v0.8 → v0.9

v0.9 introduces **OntoCore** as the platform identity. This is primarily a branding and documentation release — **no breaking API changes** to `ontoindex-*` crates, the `ontoindex` CLI, or LSP wire format.

## What changed

| Area | v0.8 | v0.9 |
|------|------|------|
| Platform brand | OntoIndex | **OntoCore** |
| Public Rust façade | — | **`ontocore` crate** on crates.io |
| Implementation crates | `ontoindex-*` | `ontoindex-*` (unchanged) |
| CLI binary | `ontoindex` | `ontoindex` (unchanged) |
| LSP binary | `ontoindex-lsp` | `ontoindex-lsp` (unchanged) |
| LSP methods | `ontoindex/*` | `ontoindex/*` (unchanged) |

Older documentation may still say "OntoIndex" — that refers to the same engine now called OntoCore.

## Rust library users

You can continue using `ontoindex-*` crates directly. Optionally adopt the façade:

```toml
[dependencies]
ontocore = "0.9"
```

```rust
use ontocore::workspace::Workspace;

let ws = Workspace::open("./ontology")?;
```

`Workspace` is **experimental** until v0.10.

## Extension (VS Code)

1. Update to **OntoCode 0.9.0** (Marketplace or release VSIX).
2. No settings or command changes required.
3. Documentation moved to `docs/ontocode/` on Read the Docs.

## CI pins

```bash
cargo install ontoindex-cli --locked --version 0.9.0
```

Or pin `ontocore = "0.9"` / `ontoindex-catalog = "0.9"` in `Cargo.toml`.
